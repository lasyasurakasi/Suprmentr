Client-Server Architecture

By Pranav J

Introduction

The client-server architecture is a computing model where the client (user's browser) sends requests to a server, and the server responds with the requested resources. This model is widely used on the internet to deliver web pages and services.

Diagram

Client sends request → Server processes → Server sends response

Explanation of Client-Server Architecture

  • Client: The user's device or browser (Chrome, Edge, etc.)
  • Server: A system that stores and serves website data
  • Internet: Medium through which communication happens
  • Request: Sent by client using HTTP/HTTPS
  • Response: Sent by server containing HTML, CSS, JS

What Happens When You Open a Website

  1. User enters a URL in the browser
  2. DNS converts the domain name into an IP address
  3. Browser sends an HTTP/HTTPS request to the server
  4. Server receives and processes the request
  5. Server sends back website files (HTML, CSS, JavaScript)
  6. Browser renders the webpage for the user
